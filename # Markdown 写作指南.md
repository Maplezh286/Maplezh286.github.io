## 什么是 Markdown？

Markdown 是一种轻量级标记语言，由约翰·格鲁伯（John Gruber）在 2004 年创建。它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的 HTML 文档。

简单了解语法即可，知道怎么调用标题，换行，强调，代码块等等最核心的东西就可以使用了。Obsidian上支持页面编辑Markdown，甚至不用学习MarkDown语法都可以写出出色的MarkDown文章。大概一个小时了解一下MarkDown就可以上手用了。

那么你可能会问：为什么我要用MarkDown呢？因为MarkDown非常强大，可以导出为HTML网页，可以导出为PDF文件，可以导出为Word文档（需要Pandoc）。而且最重要的是：比写Word文档舒服。
## 基础语法

### 标题

使用 `#` 符号表示标题，数量表示级别：

```markdown
# 一级标题
## 二级标题
### 三级标题
```

复制

### 段落和换行

段落之间用空行分隔。要创建换行，在行尾添加两个空格。

### 强调

```markdown
*斜体* 或 _斜体_
**粗体** 或 __粗体__
***粗斜体*** 或 ___粗斜体___
```

复制

### 列表

无序列表使用 `-`、`+` 或 `*`：

```markdown
- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2
```

复制

有序列表使用数字：

```markdown
1. 第一步
2. 第二步
3. 第三步
```

复制

### 链接和图片

```markdown
[链接文字](https://example.com)
![图片描述](image.jpg)
```

复制

### 代码

行内代码使用反引号：

```markdown
这是 `code` 示例
```

复制

代码块使用三个反引号：

````markdown
```javascript

    console.log("Hello, World!");
````

## 进阶技巧

### 表格

```markdown
| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 内容 1 | 内容 2 | 内容 3 |
| 内容 4 | 内容 5 | 内容 6 |
```

复制

### 任务列表

```markdown
- [x] 已完成任务
- [ ] 未完成任务
- [ ] 另一个未完成任务
```

复制

### 脚注

```markdown
这是一个有脚注的句子[^1]。

[^1]: 这是脚注的内容。
```

复制

## 写作建议

1. **保持简洁** - Markdown 的精髓就是简洁，不要过度使用格式
2. **使用标题层级** - 合理使用标题层级，让文档结构清晰
3. **列表优于长段落** - 信息较多时，使用列表更易读
4. **代码块标注语言** - 这样能获得语法高亮
