## vibe coding

### 两个必需品

1. 强大的AI是必需品，体感上，Claude Code框架+智谱大模型的理解需求、代码实现、测试优化能力明显优于Trae（但是Trae免费啊，免费的还要什么自行车？），只有在有图片描述的需求时，我才会使用Trae。

2. 描述需求的能力是必需品。

我们不说：

```
新需求：在录课界面点击“查看记录”按钮，不会返回主页，而是跳出弹窗（可上下滑动查看），显示已经记录的事件列表
```

我们说：
```
提示词：实现录课界面"查看记录"弹窗功能

需求描述：在录课界面点击"查看记录"按钮时，不返回主页，而是弹出一个可上下滑动的弹窗，显示已记录的事件列表。由于是录课阶段，长视频还未录制完成，所以只显示事件记录那一刻的截屏照片，不显示10s视频片段。

实现方案：

1. 修改"查看记录"按钮点击逻辑：
    - 将原来的 navigate('/courses') 改为打开弹窗
    - 添加状态控制：const [showEventsModal, setShowEventsModal] = useState(false)
    - 按钮点击处理：onClick={() => setShowEventsModal(true)}
    - 添加日志：console.log('📋 [查看记录] 打开事件列表弹窗')
2. 创建事件列表弹窗组件：
    - 使用 Modal 或 Dialog 组件实现弹窗
    - 弹窗标题："课程事件记录" 或 "已记录事件（X个）"
    - 弹窗内容区域可滚动：style={{ maxHeight: '70vh', overflowY: 'auto' }}
    - 添加关闭按钮：右上角 X 按钮或底部"关闭"按钮
    - 点击遮罩层关闭弹窗
3. 获取事件列表数据：
    - 调用后端 API：GET /api/v1/courses/{course_id}/events
    - 在弹窗打开时（useEffect 或 onOpen）获取数据
    - 添加加载状态：const [loading, setLoading] = useState(false)
    - 添加日志：console.log('📥 [事件列表] 开始获取数据...')
4. 事件列表展示内容：
    - 每个事件卡片包含： a. 事件照片：img src={event.photo_url}，点击可放大查看 b. 时间戳：格式化为 MM:SS（如 02:15 表示2分15秒） c. 事件类型：用不同颜色标识（正面=绿色，负面=红色，中性=灰色） d. 关联学生：显示学生姓名或头像 e. 描述信息：显示 description 字段 f. 关联规约/刺激物：如果有的话显示
    - 卡片样式：类似课后反思阶段的事件列表，保持一致性
5. 时间戳格式化函数：
    - 将秒数转换为 MM:SS 格式
    - 示例：const formatTime = (seconds) => { const mins = Math.floor(seconds / 60); const secs = seconds % 60; return `${mins.toString().padStart(2, '0')}:${secs.toString().padStart(2, '0')}`; }
6. 照片放大功能（可选）：
    - 点击事件照片时打开大图预览
    - 使用 Lightbox 或自定义模态框
    - 支持关闭和缩放
7. 空状态处理：
    - 如果没有事件记录，显示提示："暂无事件记录"
    - 显示图标和文字说明
8. 加载状态：
    - 显示 Loading Spinner 或骨架屏
    - 添加日志：console.log('⏳ [事件列表] 加载中...')
9. 错误处理：
    - 如果获取失败，显示错误提示："加载失败，请重试"
    - 添加重试按钮
    - 添加日志：console.error('❌ [事件列表] 加载失败:', error)
10. 弹窗关闭逻辑：
    - 点击 X 按钮关闭
    - 点击遮罩层关闭
    - 点击"关闭"按钮关闭
    - 关闭时清理状态：setShowEventsModal(false)`
```

这些是我写的吗？显然不是，这是我的提示词助手AI帮我写的，我只需要提出需求，然后成为两个AI交流的搬运工。

### 从0搭建项目呢？

我认为最最重要的是架构。项目架构不好，一切全白费。

怎么设计架构？目前不在我的能力范围之内，不过我知道一些方法。

比如：让AI小助手帮你设计**产品需求文档（PRD）**

将你的核心需求全部告诉小助手，让它帮你设计架构。这一步恰恰是最难最核心的一步，因为你需要穷尽你的需求，不要有任何遗漏。否则当大厦建好了，你还想修地基，这也许可以办到，却十分麻烦。所以在对Claude Code给出任何提示词之前，请你花充足的时间，和小助手深度交流一下。当然，为了配合Claude Code，使用它的前端Agent：Claude作为小助手是极好的。


## 合作精神，团队至上

vibe coding，一大半的时间在等待AI生成代码。多一个人，效率几乎翻倍。因为大家都用AI，都会说中国话（英文当然更好），都会描述需求。

所以，vibe coding时代，合作是提高效率的极好方法。

进一步地，永远不要翻脸。

可能你的同伴在提交Git之后，把你花了三个小时完成的需求覆盖了。

不要生气，AI有记忆，提示词还在，20分钟就修好了。人家也不是故意的。如果你生气质问你的同伴，对大家都不好。

项目合作，一定是团队至上。
